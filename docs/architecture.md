# 系统架构

## 定位

本项目是一个面向博士课题实验的空间信息网络仿真与可视化平台，核心场景是多层 LEO 星座与 GEO 卫星、地面站、地面网络共存时的频谱、通信、网络与计算资源管理。

## 分层架构

```text
前端可视化层
Vue 3 + TypeScript + Cesium + ECharts
        |
        | REST / WebSocket
        v
后端仿真服务层
FastAPI + Pydantic + 仿真状态管理
        |
        v
算法与数据层
轨道传播、链路预算、资源分配、路由、任务卸载、实验记录
```

## 核心实体

- Satellite：卫星节点，包含轨道参数、通信载荷、计算能力和频谱配置。
- GroundStation：地面站或网关节点，包含位置、天线能力和接入约束。
- UserTerminal：用户终端，后续用于业务请求、移动性和接入决策。
- Link：通信链路，描述连接双方、距离、仰角、路径损耗、容量和干扰状态。
- ResourceState：资源状态，包含频谱、功率、路由、计算和任务队列。
- ExperimentScenario：实验场景配置，保证论文实验可复现。

## 阶段路线

### Phase 1：可视化仿真骨架

- 简化 Walker 星座生成。
- 地面站可见性判断。
- 卫星、地面站、链路可视化。
- 基础链路预算接口。

### Phase 2：频谱与干扰

- 频段、带宽、功率、波束模型。
- 同频干扰识别。
- SINR 与容量计算。
- 随机、贪心、最小干扰优先等 baseline 算法。

### Phase 3：网络与路由

- LEO-LEO、LEO-GEO、LEO-地面多类型链路。
- 动态拓扑图。
- 最短路径、最小时延、最大剩余带宽和负载均衡路由。
- 端到端时延、阻塞率、链路利用率指标。

### Phase 4：计算资源管理

- 任务模型：数据量、计算量、时延约束、优先级。
- 执行位置：本地、LEO、GEO、地面边缘、云中心。
- 通信-计算联合优化。
- 强化学习、图学习或分布式优化算法接入。
